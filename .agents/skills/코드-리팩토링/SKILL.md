# 코드-리팩토링 스킬

## 목적
- 가독성과 유지보수성을 높이는 리팩토링을 수행합니다.

## 비목적
- 요구되지 않은 기능 추가
- 범위를 벗어난 대규모 구조 변경

## 트리거 문장 예시
- "동작은 그대로 두고 구조만 정리해줘"
- "이 파일 리팩토링하고 위험 포인트 알려줘"

## 입력
- 대상 파일/범위
- 현재 문제점
- 허용 변경 범위

## 워크플로
1. 허용 변경 범위와 금지 범위를 확정합니다.
2. 코드 스멜을 분류하고 우선순위를 정합니다.
3. 최소 단위 패치로 리팩토링을 수행합니다.
4. 동작 동일성 관점에서 영향도를 점검합니다.
5. 변경 요약/위험 요소/검증 결과를 보고 형식으로 정리합니다.

## 출력 형식
- 변경 요약
- 변경 파일 목록
- 위험 요소 및 확인 포인트

## 품질 기준
- 동작 동일성 유지
- 불필요한 변경 최소화
- 스타일/네이밍 일관성

## 실패 처리 규칙
- 범위가 불명확하면 작업 전 범위를 다시 확인합니다.
- 회귀 위험이 크면 안전한 대안(분할 적용)을 제시합니다.
- 검증이 불가하면 미검증 항목을 명시합니다.

## 연동 포인트
- command: `refactor-checklist`
- hook: `pre-task-scope-check`, `pre-delivery-skill-check`
