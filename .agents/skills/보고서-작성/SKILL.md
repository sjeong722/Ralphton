# 보고서-작성 스킬

## 목적
- 정해진 형식으로 읽기 쉬운 보고서를 작성합니다.

## 비목적
- 근거 없는 주장 추가
- 데이터 조작/과장

## 트리거 문장 예시
- "이 내용을 보고서 형식으로 정리해줘"
- "회의 결과를 3분 내 읽히는 문서로 만들어줘"

## 입력
- 보고서 주제
- 대상 독자
- 핵심 데이터/근거

## 워크플로
1. 입력 누락 여부를 먼저 점검합니다.
2. 독자 기준으로 난이도와 분량을 확정합니다.
3. 제목/요약/본문/다음 액션 순서로 초안을 작성합니다.
4. 숫자/근거 출처를 문장 단위로 확인합니다.
5. 3분 내 읽기 기준으로 길이와 중복을 줄입니다.

## 출력 형식
- 제목
- 요약(3줄)
- 본문(소제목 포함)
- 다음 액션

## 품질 기준
- 숫자/근거는 명시
- 과장 금지
- 3분 내 읽기 분량

## 실패 처리 규칙
- 입력이 부족하면 필요한 항목을 먼저 요청합니다.
- 근거가 불명확하면 "확실하지 않음"으로 표시합니다.
- 충돌되는 사실이 있으면 단정하지 않고 대안을 제시합니다.

## 연동 포인트
- command: `report-draft`
- hook: `pre-delivery-skill-check`
